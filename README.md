# 社内共有業務管理システム

Google Sheets / Google Apps Script / AppSheet を活用した、社内向けの業務管理・情報共有システムです。

予定管理、電話履歴、工事予定、休み管理、車検管理、車検履歴、日報、備品修理、運転免許、資格管理、個人ToDo、お知らせ、PDF帳簿、SQL移行設計までを Google Workspace 上で一元管理できるようにしています。

現在の安定確認版は以下です。

```text
v9.7.48p61dx
```

安定確認版ファイル：

```text
github/company_management_v9_7_48p61dx_input_sheet_ids_stable_check.js
```

---

## 概要

このシステムは、社内で分散しやすい予定・電話対応・工事情報・車両管理・日報・備品修理・免許資格情報を、Google Sheets 上でまとめて管理するための業務改善システムです。

入力用シートに登録した情報を、Apps Script によって `一覧スケジュール` と `要確認一覧` に自動集約します。

また、AppSheet と連携することで、スマートフォンやタブレットからの入力にも対応できる構成にしています。

## 主な機能

### 一覧スケジュール自動生成

各入力シートの予定、対応履歴、期限情報を `一覧スケジュール` に自動集約します。

一覧スケジュールは、すべての進行中データを表示する場所ではなく、今日・近いうちに確認すべきものを表示する画面として設計しています。

表示対象例：

* 期限切れ
* 今日
* 3日以内
* 7日以内
* 30日以内
* 要確認
* 未確認
* 差戻し
* 重要

通常進行中の工事、完了済みの予定、対応済みの電話などは、原則として一覧スケジュールに残さない方針です。

---

### 要確認一覧の自動抽出

期限が近いもの、当日対応が必要なもの、重要なお知らせ、未対応の電話などを `要確認一覧` に自動抽出します。

要確認一覧は、一覧スケジュールの中でも特に確認が必要なものだけを集める画面です。

---

### 電話履歴管理

電話履歴では、電話受付、折返し、対応メモ、完了確認を管理します。

現在の運用では、旧来の `電話対応` 列ではなく、以下を中心に管理します。

* 新着
* 日付
* 時間
* 相手
* 折り返し電話番号
* 内容
* 担当
* 完了チェック
* 対応メモ
* 既読
* 個人確認
* 備考
* 電話ID

電話履歴の基本方針：

* 完了チェックONで完了扱い
* 対応中、折返し予定、確認中などは対応メモへ記入
* 対応メモありの電話は新着・一覧対象から外す
* 完了済み電話は一覧スケジュールから外す
* 電話IDは内部管理用として保持し、通常は非表示

v9.7.48p61dx では、電話IDの復旧と診断確認まで対応済みです。

---

### 工事予定管理

工事予定では、以下の情報を管理します。

* 工事名
* 現場
* 依頼主
* 連絡先
* 契約金額
* 税
* 開始日
* 終了日
* 状態
* 担当
* 通知
* 既読
* 個人確認
* 備考
* 工事ID

工事予定は、状態だけで一覧スケジュールに出すのではなく、通知がある場合に表示します。

例：

* 施工中でも通常進行中なら一覧に出さない
* 終了日が近い場合は表示
* 完了、請求済み、中止、不要、対象外は除外

---

### 休み管理

休み管理では、年休、半休、代休、欠勤などを管理します。

管理項目例：

* 申請日
* 休み開始日
* 休み終了日
* 対象者
* 区分
* 代理担当
* 状態
* 通知
* 備考
* 休みID

未来の休み、今日の休み、申請中の休みは一覧対象になります。

取得済み、完了、取消、却下、中止などは一覧から外す方針です。

---

### 見積書管理

見積書では、見積依頼、提出、返答待ち、受注、失注を管理します。

管理項目例：

* 件名
* 依頼主
* 場所
* 金額
* 税
* 提出年月日
* 工事内容
* 状態
* 通知
* 備考
* 見積ID

提出依頼や要確認の見積を一覧・要確認に出し、提出済みや返答待ちなどは必要に応じて完了扱いとします。

---

### 車検管理・車検履歴

車検管理では、車両ごとの車検期限、車検予定日、新車検期限、車検状態、担当、通知、備考を管理します。

車検更新時には、車検履歴へ記録します。

基本運用：

```text
1. 車検管理で新車検期限を入力
2. 車検状態を更新済にする
3. 車検履歴へ旧期限・新期限を記録
4. 車検期限を新しい期限へ更新
5. 車検予定日・新車検期限をクリア
6. 状態を有効または空欄へ戻す
```

車検履歴は追記専用として扱い、通常の月次整理対象からは外します。

---

### 資格管理

資格管理では、社員ごとの資格、免許、技能講習、特別教育、安全教育の保有状況を管理します。

管理項目例：

* 所有者
* 区分
* 資格名
* 取得区分
* 保有状況
* 取得日
* 更新期限
* コピー有無
* 状態
* 通知
* 備考
* 資格ID

資格管理の方針：

* 資格名は区分に応じて候補を絞る
* 候補順は資格マスタの表示順を正とする
* 手入力時に取得区分・保有状況を勝手に補完しない
* 所有者＋資格マスタ順で整理する
* 所有者ごとにグループ化する
* 月別畳みは対象外

v9.7.48p61dx では、所有者＋マスタ順整理と所有者グループ化を確認済みです。

## メニュー構成

このシステムでは、スプレッドシート上に `社内管理` メニューを追加します。

Google Sheetsのカスタムメニューは画面下でスクロールできないことがあるため、メニューを用途別に分割しています。

主な構成：

```text
社内管理
→ 日常更新（一覧＋要確認・軽量）
→ 今のシートだけ日時ソート
→ 今のシートだけセル調整
→ 今のシートだけコンパクト表示
→ 今のシートだけ既読セル修復
→ 1. 電話・休み
→ 2. 過去整理・月次
→ 3. 安全・教育
→ 4. 予定・工事・見積
→ 5. 車検・免許・資格
→ 6. 表示・帳票
→ 7. 保守・チェック
```

社内管理

## 電話履歴の運用

電話履歴では、完了チェック方式を使います。

```text
社内管理
→ 1. 電話・休み
→ 電話：完了チェック順に整理
```

確認済みの動作：

* 未完了が上
* 完了が下
* 完了済みは一覧から外れる
* 対応メモありは新着・一覧から外れる

---

## 資格管理の運用

資格管理では、以下の順で整理します。

```text
社内管理
→ 5. 車検・免許・資格
→ 資格：整理・一覧
→ 所有者＋マスタ順に整理
```

その後、必要に応じて所有者ごとにグループ化します。

```text
社内管理
→ 5. 車検・免許・資格
→ 資格：表示（軽量）
→ 所有者ごとにグループ化（軽量）
```

## 過去整理

完了済みデータは、原則として本体シートから物理移動しません。

本体シートに残したまま、過去整理=済 にして通常一覧から外す方針です。

過去整理用の共通列：

* 対象月
* 過去整理
* 過去整理日時
* 復元メモ

ただし、v9.7.48p61dx 時点では、月別畳み処理は起動時間超過があるため保留しています。

## 実装上の工夫

### 設定シート管理

担当者、既読確認者、社用車、修理業者などを設定シートで管理し、コードを直接編集しなくても運用設定を変更できるようにしています。

---

### 入力済み行だけチェックボックス表示

空行に既読チェックボックスが大量に残ると、見づらくなるだけでなく getLastRow が伸びて重くなる原因になります。

そのため、入力済み行だけに既読・個人確認チェックボックスを表示するようにしています。

---

### ID列によるAppSheet対応

AppSheetで `_RowNumber` をKeyにしないよう、各入力シートにID列を用意しています。

v9.7.48p61dx では、入力シートIDを一括作成する軽量メニューを追加しています。

---

### 電話履歴の完了チェック方式

電話履歴は、状態プルダウン中心ではなく、完了チェックと対応メモを中心に運用します。

これにより、空欄に意味がある列でプルダウン候補が誤入力される問題を避けています。

---

### 保守メニューの隔離

普段使うメニューと、重い処理・危険な処理を分けています。

通常運用中に誤って全体更新や重い修復を実行しにくい構成にしています。

---

### 軽量診断

起動時間超過を避けるため、診断用シート作成は軽量版にしています。

確認内容は、シート存在、必須ヘッダー、主要列、一覧・要確認件数、電話履歴、資格管理、AppSheet確認項目に絞っています。
